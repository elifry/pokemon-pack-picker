Card recognition setup (optional)

When enabled in Settings, a camera button appears on each slot so you can scan cards and track what you pulled. No images are sent to third parties; they go only to the local service URL you configure.

Requirements

A local HTTP service that accepts a card image and returns a card identifier (e.g. Pokemon TCG API card id like swsh12-123).

The app POSTs to {your URL}/recognize with multipart form data: field name image, JPEG or PNG. The service should return JSON: {"cardId": "setX-nnn"} or {"card_id": "setX-nnn"}.

Self-hosted options

Run a pipeline that (1) detects the card in the image (e.g. YOLO), (2) identifies it (e.g. ResNet50 vs official card images), (3) returns the Pokemon TCG API card id.

Example: lo-calvin/Trading-Card-Scanner (Python, YOLOv11 + ResNet50). Expose a small HTTP endpoint that accepts an image and returns {"cardId": "..."}.

Configuring the app

  1. Start your recognition service (e.g. http://127.0.0.1:5000).
  2. In Settings, check Enable card recognition.
  3. Set Recognition service URL to the base URL (e.g. http://127.0.0.1:5000). The app will POST to {url}/recognize.

If the URL is empty, tapping the camera button will show a message that recognition isn’t set up and offer to disable the feature.
